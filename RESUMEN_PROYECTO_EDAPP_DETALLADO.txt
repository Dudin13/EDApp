1. IDENTIDAD Y PROPÓSITO
=========================
EDApp es una plataforma de análisis de vídeo de fútbol avanzada, desarrollada 
específicamente para el Departamento de Datos del Cádiz CF Cantera. 
Su objetivo es automatizar la extracción de métricas de rendimiento (posicionales, 
eventos, estadísticas) a partir de grabaciones de partidos (principalmente cámaras VEO).

Tecnologías Core:
- Lenguaje: Python 3.10+
- Interfaz: Streamlit
- IA/ML: YOLO11 (Detección/Segmentación), ByteTrack (Seguimiento), SAM2 (Etiquetado)
- Gestión de Datos: DVC (Data Version Control)
- Entorno: CUDA 12.1 con soporte para GPU dedicada (RTX 2070 SUPER o superior)

2. ARQUITECTURA DEL PIPELINE (Estado de Módulos)

3. ESTADO DEL MODELO Y ENTRENAMIENTO (ML)
=========================================
El modelo actual de referencia es 'players_v3' (basado en YOLO11m-segmentation).

Rendimiento (v3):
- mAP50 (Box): 0.818
- mAP50 (Mask): 0.796 (aprox)
- Métrica por clase:
  - Players: ~0.93 mAP50
  - Referees: ~0.94 mAP50
  - Goalkeepers: ~0.85 mAP50
  - Ball: ~0.55 mAP50 (Área de mejora principal)

Dataset Actual (Hybrid Dataset):
- Total Imágenes: ~2,930 (2,340 Train / 590 Val)
- Clases (4): 0: player, 1: goalkeeper, 2: referee, 3: ball.
- Origen: Mezcla de datos propios del Cádiz CF (VEO), Roboflow y SoccerNet.

Configuración Crítica de Inferencia:
- imgsz=1280: Esencial para detectar jugadores en cámaras VEO (gran angular), 
  donde los jugadores son muy pequeños. Con 640 la mayoría son ignorados.
